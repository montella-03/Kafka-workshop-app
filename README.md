# kafka-transaction-demo

Production-ready Spring Boot 4 + Kafka + Schema Registry demo using Avro events, Conduktor, and full observability with Prometheus and Grafana.

## Stack

- Java 21
- Spring Boot 4.0.3
- Spring Kafka + Confluent Schema Registry (Avro)
- Maven
- Docker Compose

## What It Does

- `POST /api/transactions` accepts transaction JSON and publishes `TransactionEvent` to topic `transaction-events`.
- `TransactionEventProcessor` consumes `transaction-events` and publishes `TransactionReceivedEvent` to `transaction-received-events`.
- `TransactionReceivedConsumer` consumes `transaction-received-events` and logs received events.
- Avro schemas are in `src/main/resources/avro/` and Java classes are generated by Maven.

## Run Everything

```bash
docker compose up --build
```

## Access

- Conduktor: http://localhost:8080
- Grafana: http://localhost:3000 (admin/admin)
- Prometheus: http://localhost:9090
- Swagger UI: http://localhost:8082/swagger-ui.html
- OpenAPI JSON: http://localhost:8082/v3/api-docs
- Actuator Prometheus: http://localhost:8082/actuator/prometheus

## Send a Test Transaction

```bash
curl -X POST http://localhost:8082/api/transactions \
  -H "Content-Type: application/json" \
  -d '{
    "transactionId": "txn-1001",
    "userId": "user-42",
    "amount": 149.99,
    "currency": "USD",
    "timestamp": "2026-03-01T20:00:00Z"
  }'
```

## Validate Events and Metrics

1. Open Conduktor and select `Local Kafka` cluster.
2. Check topic `transaction-events` for submitted events.
3. Check topic `transaction-received-events` for processor output.
4. Open Grafana and view:
   - `Spring Boot JVM + Kafka Client Metrics`
   - `Kafka Overview (ID 7589 Similar)`
5. Open Prometheus and verify targets are `UP` in `Status -> Targets`.
